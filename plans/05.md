# README-Only Surface Reduction

## Summary

- Use [README.md](../README.md) as the sole supported contract and apply a hard and aggressive prune to undocumented operational helpers, extension hooks, and package conveniences.
- Keep only these supported entrypoints after the refactor:
  - `ociapp.Application`
  - `ociapp serve --app <pkg.module:attr>`
  - `ociapp-build <project_root> [--output-dir <dir>]`
  - `[tool.ociapp-build]` keys `mode`, `entrypoint`, `system-packages`, and `containerfile`
  - `ociapp_runtime.Runtime` as an async context manager with `execute(...)`
  - `Runtime` kwargs `engine`, `startup_timeout`, `request_timeout`, `shutdown_timeout`, `idle_timeout`, and `reaper_interval`
  - `ociapp_runtime.DockerAdapter()` as the default engine
- Baseline is green before the reduction work:
  - `timeout 180s uv run pytest -q`
  - `timeout 180s uv run mypy .`

## Key Changes

- `ociapp`
  - Reduce [packages/ociapp/src/ociapp/__init__.py](../packages/ociapp/src/ociapp/__init__.py) to export only `Application`.
  - Keep `ociapp serve --app` and remove the undocumented `--socket-path` flag from the CLI; make the socket path a private implementation constant.
  - Make loader, server, and wire helpers implementation-only: stop treating `load_application`, `OciAppServer`, `serve_application`, protocol encode/decode helpers, payload helpers, envelope models, and package-root error re-exports as supported APIs.
  - Delete dead exception types `FrameError` and `EnvelopeError`, which are defined but unused.

- `ociapp_runtime`
  - Reduce [src/ociapp_runtime/__init__.py](../src/ociapp_runtime/__init__.py) to export only `Runtime` and `DockerAdapter`.
  - In [src/ociapp_runtime/runtime.py](../src/ociapp_runtime/runtime.py), remove undocumented constructor args `runtime_root` and `clock`, and remove public `start()` and `close()` from the supported surface; lifecycle should live behind `__aenter__` and `__aexit__`.
  - Make transport and session helpers internal-only: `WorkerSession`, `open_worker_session`, `execute_request`, `EngineAdapter`, `InstanceState`, runner types, and package-root error conveniences are no longer supported imports.
  - Remove the runtime’s dependency on `ociapp` package-root convenience imports so the reduced `ociapp.__init__` stays minimal.

- `ociapp-build`
  - Treat `ociapp-build` as CLI-only at the supported surface; remove the supported package-root Python API from [packages/ociapp-build/src/ociapp_build/__init__.py](../packages/ociapp-build/src/ociapp_build/__init__.py).
  - Route the CLI through private build and config helpers and private command execution hooks instead of public dataclasses, runners, and helper functions.
  - Stop treating `build_project`, `load_build_project`, `BuildProject`, `ManagedBuildConfig`, `CustomBuildConfig`, `CommandRunner`, `build_image_tag`, `resolve_artifact_path`, `build_wheel`, and `prepare_managed_context` as supported imports.
  - Keep the current CLI parser shape, including optional `--output-dir`, but do not preserve any Python-level compatibility for removed build helpers.

- Docs and examples
  - Align mutable docs and examples with the root README after the API reduction, especially [example/README.md](../example/README.md), which currently shows stale build invocation syntax.

## Test Plan

- Remove tests that currently lock in removed APIs: package-root `__all__` breadth, `--socket-path`, `Runtime.start()` and `Runtime.close()`, `runtime_root`, `clock`, `DockerAdapter(runner=...)`, runtime transport helpers, and `ociapp-build` Python helpers.
- Keep coverage on the retained contract:
  - `Application` request and response validation
  - `ociapp serve --app`
  - managed and custom build config parsing
  - `ociapp-build` CLI invocation
  - `Runtime` defaulting to `DockerAdapter`
  - request timeout, startup timeout, and idle reaping behavior
  - warm-worker reuse
  - concurrent execution on one worker
  - `Runtime.execute()` success and error paths
- Run the required verification after implementation:
  - `timeout 180s uv run pytest`
  - `timeout 180s uv run mypy .`
  - `timeout 300s uv run prek run -a`

## Assumptions

- The current root README is authoritative even though the worktree is dirty; `example/README.md` and package READMEs are not contract sources.
- README TODO items about future executors and engines do not justify keeping today’s undocumented extension hooks public.
- The README documents supported flags and constructor knobs, not every optional omission, so `ociapp-build` may keep its existing default output-directory behavior even under the hard-prune plan.
